# Code of Conduct

Gromaq welcomes contributors who help build a correct, fast, native terminal
with clear evidence and respectful technical discussion.

## Expected Behavior

- Be direct, factual, and respectful.
- Critique code, claims, tests, and tradeoffs rather than people.
- Keep compatibility, performance, and security reports reproducible.
- Respect maintainers' requests for minimized examples, proof boundaries, and
  issue templates.
- Avoid posting private data, secrets, exploit details, or crash material that
  belongs in a private security report.

## Unacceptable Behavior

- Harassment, threats, insults, or discriminatory language.
- Repeatedly derailing technical threads after maintainers have redirected the
  discussion.
- Publicly posting vulnerability details that should follow `SECURITY.md`.
- Misrepresenting benchmark, compatibility, or test evidence.

## Enforcement

Maintainers may edit, hide, lock, or remove comments and issues that violate
these expectations. Repeated or severe violations may lead to temporary or
permanent loss of participation privileges in project spaces.

Security reports should follow [`SECURITY.md`](SECURITY.md).
