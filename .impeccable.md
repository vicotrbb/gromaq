## Design Context

### Users

Gromaq is for developers, terminal-heavy operators, and open-source contributors
who spend long sessions in shells, editors, multiplexers, and log-heavy command
workflows. The core job is daily terminal use: typing commands, reading dense
output, switching through shell/editor/tmux contexts, and trusting that the
native renderer stays fast and correct.

### Brand Personality

Precise, calm, and capable. The product should feel like a serious native tool:
fast, readable, stable, and carefully tuned rather than decorative. It should be
comfortable enough for all-day use and polished enough for public screenshots.

### Aesthetic Direction

Dark mode is the default because terminal usage commonly happens in focused
developer sessions and long-running command workflows. The visual target is
inspired by Ghostty and carefully configured iTerm2 setups: clear text,
controlled contrast, subtle tinted dark surfaces, expressive ANSI colors,
visible cursor/selection states, and enough padding to avoid a cramped terminal
without wasting working space.

### Design Principles

- Prioritize readability over ornament: text, cursor, selection, and ANSI colors
  must remain legible at normal laptop and desktop viewing distances.
- Keep the default theme excellent out of the box, then expose configuration
  building blocks for users who want their own terminal identity.
- Use color with purpose: neutral dark surfaces, rare bright accents, and
  semantic terminal colors that remain usable in real shells and TUIs.
- Preserve native terminal discipline: no web UI patterns, no decorative
  gradients, no generic card-heavy layout language, and no visual choices that
  compromise frame time or renderer correctness.
- Validate visuals with deterministic config, contrast, pixel, preview, and
  runtime smoke tests before treating them as complete.
