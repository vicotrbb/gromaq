# Architecture

Gromaq separates terminal correctness from platform integration.

## Core

The core library owns deterministic terminal state:

- `cell`: cell and style representation
- `app`: native `winit` application lifecycle, window attributes, and event-loop handler boundary
- `grid`: fixed-size visible grid
- `scrollback`: bounded styled-cell history with resize reflow
- `dirty`: coalesced dirty-region tracking for renderer scheduling
- `selection`: visible-grid selection ranges and copy extraction
- `clipboard`: host clipboard abstraction plus deterministic in-memory and native OS text adapters
- `terminal`: parser-backed terminal state engine
- `input`: keyboard-to-byte encoding for deterministic test keys and native `winit` logical keys
- `mouse`: DEC mouse reporting state and SGR event encoding
- `font`: Swash-backed font-byte loading, glyph rasterization to RGBA8 atlas bitmaps, and renderer-plan glyph bitmap caching
- `native_gpu`: native `wgpu` adapter/device bootstrap and adapter diagnostics
- `config`: validated user configuration
- `test_api`: structured debug and validation API with deterministic performance counters and cell screenshots

The core is headless and tested without a window or GPU. This keeps terminal behavior reproducible. Test API performance counters expose parser bytes, dirty cells, dirty-region batches, scrolls, and resizes for deterministic validation. Test API screenshots are deterministic one-pixel-per-cell RGBA captures of terminal state; live GPU/window screenshot proof remains part of the native rendering validation gap.

## PTY Boundary

`pty` defines shell launch and PTY sizing around `portable-pty`. Process lifecycle integration is intentionally kept separate from terminal state so PTY tests can validate shell behavior without coupling to rendering. `PtySession` can start a background reader, drain already-available output without consuming the session lifecycle path, notify a wakeup callback when output bytes are queued, write input bytes, and resize the retained native PTY master. The native app runtime builds terminal state from startup dimensions, starts the configured shell PTY once after window creation, retains the session handle for the lifetime of the app runtime, drains PTY output into the terminal parser, maps native window resize events to terminal dimensions, resizes terminal state and the retained PTY session, encodes native keyboard events, committed platform text, terminal paste payloads, and terminal mouse reports, maps native window mouse coordinates to grid cells, and writes input bytes back to the PTY. The native event loop schedules bounded timer wakeups for PTY polling, also accepts PTY output-ready user events from the background reader through a clonable event proxy, and requests redraw only when pumped output changes terminal state.

## Renderer Boundary

`native_gpu` creates a real `wgpu::Instance`, requests a high-performance adapter, creates a device and queue, exposes stable adapter metadata for diagnostics, can safely create a window surface handoff with adapter-specific capabilities, can run an offscreen render-pass clear with RGBA8 readback, can upload RGBA8 texture bytes then read them back for verification, can draw a sampled textured quad into an offscreen render target with readback, and can draw terminal-planned real-font glyph quads through the same sampled GPU path. `renderer` defines the GPU renderer interface, consumes dirty-region inputs from the terminal core, includes a deterministic frame scheduler for 144Hz pacing decisions, owns a deterministic glyph atlas cache with hit/miss/eviction metrics, builds CPU-side render plans from grid snapshots with a configured font size embedded in glyph cache keys, converts planned glyphs and atlas slots into textured quads plus triangle indices for vertex/index buffer upload, prepares owned surface glyph frames from render plans plus rasterized glyph bitmaps, can pack fixed-size glyph bitmaps into a deterministic RGBA8 atlas image, can choose a deterministic low-latency `wgpu::SurfaceConfiguration` from surface capabilities, tracks surface lifecycle transitions for configure, resize reconfigure, and zero-size deferral, applies configure/reconfigure transitions to a surface backend, and includes a presentable `WgpuSurfaceBackend` path that acquires the current surface texture, clears it through a render pass, uploads a supplied terminal glyph atlas, draws supplied glyph quad batches, submits commands, and presents the frame. `app` owns a `NativeWindowSurface` wrapper that keeps the concrete surface backend, capabilities, and configuration controller together for initial configure, resize, zero-size suspension, and present-frame delegation, discovers a default native monospace font for an app-owned rasterized glyph cache, and wires dirty native redraws from runtime terminal snapshots through configured render planning, glyph rasterization, prepared surface glyph frames, and surface presentation. `font` can load real font bytes through `swash`, map a character to a glyph, rasterize it, return a dense RGBA8 bitmap suitable for atlas packing, and cache the distinct glyph bitmaps required by a render plan, including cached bitmaps needed by repeated frames. The text-atlas smoke path builds a terminal frame, plans glyphs, rasterizes distinct real-font atlas entries, packs the resulting text atlas, uploads it through `wgpu`, and reads it back byte-for-byte. The terminal-text draw smoke path turns that same frame into glyph quads, draws them into an offscreen render target, and verifies non-transparent output pixels by readback. `WgpuRenderer` records the last planned frame and can plan either dirty regions or a full viewport based on configuration. Live desktop screenshot/runtime proof of windowed terminal text drawing is upcoming work.

## Native App Boundary

`app` defines `NativeAppConfig`, deterministic `NativeAppLifecycle`, `NativeAppEvent`, `NativeAppEventProxy`, `NativeMouseGridMapper`, `NativeResizeGridMapper`, `NativeWindowSurface`, `NativeTerminalRuntime`, `NativeRuntimePerfSnapshot`, `NativeTerminalApp`, `run_native_app`, `run_native_app_with_runtime_config`, and explicit runtime/renderer configuration entry points. Validated `GromaqConfig` values can be mapped into native app frame targets, runtime terminal dimensions, scrollback limits, shell command settings, renderer dirty-region mode, and rounded renderer font size before constructing a native app handler. The `winit` `ApplicationHandler` creates a native window on resume, creates and configures a real `wgpu` surface for that window, starts the configured shell PTY through the runtime, pumps PTY output during `about_to_wait` and on PTY output-ready user events, schedules the next PTY poll with `ControlFlow::WaitUntil`, requests redraws only after terminal output is applied, maps native resize events to terminal/PTY sizes and surface reconfiguration, tracks modifier changes and cursor position, writes pressed keyboard input and committed platform text to the PTY, maps native paste shortcuts through the host clipboard and terminal paste encoder, syncs terminal-owned clipboard text through the host clipboard abstraction, maps mouse button and wheel input to terminal mouse reports, renders dirty terminal snapshots through `WgpuRenderer` on redraw events, rasterizes planned glyphs through an app-owned font cache, presents prepared glyph frames to the configured window surface, records redraw events, and exits on close/destroy events. `NativeWindowSurface` keeps a concrete surface backend, surface capabilities, and the renderer surface configuration controller together so app-owned state applies initial configure and resize actions exactly once per required surface transition and delegates clear-frame or supplied glyph-frame presentation to presentable backends. `NativeTerminalRuntime` can pump drained PTY output into terminal state, sync OSC 52 clipboard text to a host clipboard adapter, render dirty terminal frames through the renderer boundary, resize terminal state plus the retained PTY session, send encoded native key, committed text, clipboard paste, explicit paste, mouse-report, window-coordinate mouse, and raw input bytes to the PTY, expose deterministic runtime counters for PTY output/input, terminal responses, resize events, render attempts, rendered frames, clean-frame skips, bounded render-time p95, and bounded app-input-to-render latency p95, and emit structured `tracing` diagnostics for startup, PTY pump/input, render, and clean-frame skip events. The terminal parser handles OSC title updates and bounded OSC 52 clipboard-text decoding into terminal state. `cli` dispatches the no-argument binary path and `--config <path>` into this native app launcher, while smoke commands remain injectable for deterministic tests. This proves launch-path wiring, validated config-file app/runtime/renderer launch wiring, native event-loop lifecycle state, shell PTY session startup, timed PTY pump scheduling, PTY output-ready event proxy wakeups, native keyboard/text-to-PTY input, runtime resize-to-PTY notification, native window resize mapping, safe GPU window-surface handoff creation, app-owned surface configure/resize/clear-present state, supplied glyph-frame presentation delegation, redraw-to-renderer planning, default native monospace font cache loading, redraw-to-prepared-glyph-frame presentation wiring, runtime clipboard-paste/mouse-report-to-PTY input, native mouse coordinate mapping, OSC 52 payload decoding and host-clipboard sync through the abstraction, and runtime PTY I/O application, not live desktop screenshot/runtime proof of windowed terminal glyph drawing, hardware-backed frame pacing, live OS clipboard smoke, or live tmux/editor mouse workflows.

## CLI

`gromaq --gpu-info` attempts native GPU bootstrap and prints adapter metadata. `gromaq --gpu-smoke` clears a 4x4 offscreen texture through a render pass, copies it to a buffer, maps the buffer, and reports the first pixel plus non-zero byte count. `gromaq --gpu-upload-smoke` uploads a deterministic 2x2 RGBA8 texture and verifies readback byte-for-byte. `gromaq --gpu-glyph-atlas-smoke` packs two deterministic glyph-slot bitmaps into an atlas texture, uploads it, and verifies readback byte-for-byte. `gromaq --gpu-text-atlas-smoke` builds a tiny terminal text frame, rasterizes distinct planned glyphs from a real system font, packs the atlas, uploads it, and verifies readback. `gromaq --gpu-textured-quad-smoke` uploads a deterministic 2x2 texture, samples it through a full-target quad render pipeline, and reads back the rendered target. `gromaq --gpu-terminal-text-smoke` builds terminal state, plans glyphs, rasterizes a real-font atlas, converts planned glyphs into quads, draws them through `wgpu`, and reads back the target. `gromaq --config <path>` loads a TOML config file and launches the native app with validated frame-target, terminal-dimension, scrollback, dirty-region, and rounded renderer-font-size settings. `gromaq --config-check <path>` loads a TOML config file, applies defaulted partial sections, validates bounded terminal/font/performance settings, and reports the resolved values without GPU bootstrap. `gromaq --runtime-clipboard-paste-smoke` seeds the host clipboard, routes it through `NativeTerminalRuntime::send_clipboard_paste`, verifies the pasted bytes reached the retained PTY session, and restores the prior clipboard text without GPU bootstrap. `gromaq --runtime-glyph-frame-smoke` pumps deterministic runtime output, plans a dirty terminal frame, rasterizes glyphs from the default native font stack, prepares an owned surface glyph frame, and reports quads, atlas bytes, and frame dimensions without GPU bootstrap. `gromaq --runtime-perf-smoke` sends a deterministic native key through an echo PTY session, pumps output into terminal state, renders a dirty frame through the CPU-side renderer boundary, and reports structured render/input latency counters without GPU bootstrap. `gromaq --runtime-large-output-smoke` pumps a deterministic 512-line PTY burst into terminal state, verifies scrollback is capped to 128 lines without retaining the first generated line, renders the resulting dirty frame through the CPU-side renderer boundary, verifies the last generated line is visible in the render plan, and reports pumped bytes plus render counters without GPU bootstrap. `gromaq --runtime-bounded-state-smoke` pumps four deterministic long-output batches through the runtime while proving capped scrollback and dirty-frame rendering stay bounded. `gromaq --runtime-alternate-screen-smoke` pumps primary, alternate-screen, and restore output stages through the runtime, renders each dirty stage, verifies primary grid restoration, and verifies alternate-screen output stays out of scrollback. `gromaq --frame-scheduler-smoke` simulates a deterministic 144Hz dirty-frame timeline and reports target interval, frame-paced wait, presented-frame count, and dropped-frame count without GPU bootstrap. Local runs on 2026-06-22 proved `Apple M1 Pro` through the `Metal` backend, a clear-pass readback pixel of `[26, 51, 77, 255]`, texture upload readback matching `16/16` bytes, deterministic glyph atlas upload readback matching `32/32` bytes, text atlas upload readback matching `1144/1144` bytes for a 22x13 atlas with 148 covered pixels, textured-quad draw readback with first pixel `[255, 0, 0, 255]` plus 16 drawn pixels in a 4x4 target, and terminal text draw readback with 3 glyphs, 3 quads, 2 rasterized glyphs, 1 reused glyph, and 212 drawn pixels in an 88x26 target. This proves offscreen GPU command submission, readback, generic texture upload, deterministic atlas upload, real-font terminal text atlas upload, a minimal textured draw pipeline, an offscreen terminal text draw pipeline, deterministic config-file validation, deterministic config-file native app/runtime/renderer launch wiring, deterministic runtime clipboard-paste routing, deterministic runtime glyph-frame preparation, deterministic runtime performance-counter plumbing, deterministic runtime large-output render plumbing, deterministic runtime bounded-state render plumbing, deterministic runtime alternate-screen restore/render plumbing, and deterministic frame-scheduler decisions. Font rasterization is separately proven by `tests/font_rasterizer.rs`; renderer-plan glyph bitmap population is proven by `tests/rasterized_glyph_cache.rs`; native redraw-to-glyph-frame presentation wiring is proven by `tests/app.rs`; live desktop screenshot/runtime proof of the windowed terminal text renderer is not yet proven.

## Hot-Path Rules

- Parser and grid updates must remain deterministic.
- Rendering must consume snapshots and dirty regions rather than mutate terminal state.
- PTY reads must not block input or render hot paths.
- Optimizations require tests or benchmarks before claims are made.
